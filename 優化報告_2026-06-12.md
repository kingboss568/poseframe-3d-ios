# PoseFrame Studio 優化報告（2026-06-12）

## 一、修了什麼

### 🔴 付費解鎖修復（最關鍵）
**原因找到了：Paywall sheet 掛在主畫面，但 3D 編輯器是 fullScreenCover 蓋在上面，所以在編輯器內點任何 Pro 內容，購買頁永遠彈不出來。**

| 檔案 | 修改 |
|---|---|
| `Views/PoseEditorView.swift` | 編輯器內新增 Paywall sheet（編輯器情境專用） |
| `Views/ExportSheet.swift` | 匯出頁內新增 Paywall sheet（匯出情境專用） |
| `Views/ContentView.swift` | 主畫面 Paywall 改為條件式 binding，三處不互搶 |
| `State/AppState.swift` | PremiumStore 加 UserDefaults 解鎖快取（離線/冷啟動立即顯示已解鎖）、退款自動撤銷解鎖、恢復購買改為權威查詢、錯誤訊息更明確 |
| `Views/ContentView.swift` | 購買頁開啟時自動載入商品、商品載入失敗顯示「重新載入價格」按鈕、按鈕顯示實際價格 |

### 3D 操作優化
| 問題 | 修法 |
|---|---|
| SceneKit 內建相機和 App 相機狀態互相打架 | 移除 `allowsCameraControl`，自寫手勢：單指拖曳旋轉、捏合縮放、雙指上下調視線高度、雙擊回正。所有手勢同步回 state，預設視角按鈕/滑桿不再失效 |
| 完全不能調姿勢 | 新增「關節」面板：點模型部位直接選取關節（自動跳到關節面板＋震動回饋），拖曳旋轉，或用 X/Y/Z 滑桿微調，可單關節重置/全部還原 |
| 焦距滑桿無效 | 移除蓋掉 focalLength 的 fieldOfView 寫入；刪掉重複的「透視」滑桿，新增「高度」滑桿 |
| 相機俯仰只是傾斜鏡頭不是環繞 | 改為正規 orbit rig（yaw→pitch→distance），俯視/仰視真的會繞著人物轉 |

### 3D 模型擬真度
| 問題 | 修法 |
|---|---|
| USDZ 原始貼圖被整隻塗成單色 lambert | **保留原始材質**；只有剪影模式才塗黑，關閉時自動還原 |
| 姿勢強度固定 0.34，模型永遠接近 T-pose | 改為可調「姿勢強度」滑桿（預設 0.65），在關節面板 |
| T-pose 模型手臂像稻草人 | 新增自動校正：載入時試轉量測前臂高度，自動找出讓手臂自然下垂的旋轉方向（不依賴骨架軸向慣例） |
| 沒有真陰影 | 主光啟用 castsShadow（2048 shadow map、柔邊）；「顯示陰影」現在控制真陰影，不再是藏地板 |
| 效能 | 抗鋸齒 4X→2X、關閉連續渲染與 jittering、剪影材質只在切換時套用（原本每幀全模型掃一次） |

## 四、風險與備案
- **T-pose 自動校正**與**姿勢強度 0.65** 在不同 USDZ 上效果可能不同 → 已做成滑桿，使用者可自行調；若某模型異常，把 `calibrateRestPose()` 的 `dropRatio < 0.55` 門檻調低即可停用該模型的校正
- 解鎖快取使用 UserDefaults（key: `poseframe.pro.unlocked`），越獄機可改值，但一次性買斷 App 風險可接受；要更嚴可改 Keychain
